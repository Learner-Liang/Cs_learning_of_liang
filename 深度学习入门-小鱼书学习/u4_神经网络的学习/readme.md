# 第四章 神经网络的学习

本目录对应《深度学习入门》中“神经网络的学习”一章。本章的核心问题是：如果已经有了一个神经网络结构，那么参数 `W` 和 `b` 应该如何从数据中自动学出来？

与第三章只做前向推理不同，这一章开始引入损失函数、数值微分、梯度和参数更新，从而让网络真正具备“学习”的能力。

本目录中的主要代码如下：

- `ch1损失函数.py`：实现交叉熵误差，并演示 mini-batch 取样。
- `2. 数值微分.ipynb`：用 notebook 形式演示数值微分、梯度与梯度下降。
- `ch4function.py`：整理常用函数，包括 `sigmoid`、`softmax`、交叉熵以及数值梯度函数。
- `ch3两层神经网络.py`：用数值微分训练两层神经网络。
- `model_two_layer.npz`：保存训练中的网络参数与损失记录。

## 本章核心思想

神经网络学习的目标，是找到一组参数，使模型在训练数据上的预测尽可能接近真实标签。这个过程一般包括以下几个步骤：

1. 用前向传播得到预测结果
2. 通过损失函数衡量预测和真实标签之间的差异
3. 计算损失函数对参数的梯度
4. 按梯度下降方向更新参数

因此，本章围绕以下几个关键词展开：

- 损失函数
- mini-batch
- 数值微分
- 梯度
- 梯度下降

## ch1 损失函数

### 实验目的

理解为什么神经网络学习不能只看“准确率”，而需要引入一个可以连续优化的损失函数。

### 实验原理

#### 1. 为什么不能直接把准确率当作优化目标

准确率是离散变化的。即使参数发生很小改变，准确率也常常完全不变，这样就很难利用梯度信息来更新参数。

相反，损失函数会随着输出概率的变化连续变化，因此更适合作为优化目标。

#### 2. 交叉熵误差

当前代码实现了两种形式的交叉熵：

- `cross_entropy_error(y, t)`：适用于 one-hot 标签
- `false_cross_entropy_error(y, t)`：适用于标签是类别索引的情况

对于 one-hot 标签，交叉熵可写为：

`L = - sum(t * log(y + 1e-7)) / batch_size`

其中：

- `y` 是模型输出概率
- `t` 是真实标签

加入 `1e-7` 是为了避免 `log(0)` 带来的数值问题。

#### 3. mini-batch 学习

在 [ch1损失函数.py](e:/my_github/Cs_learning_of_liang/深度学习入门-小鱼书学习/u4_神经网络的学习/ch1损失函数.py#L31) 到 [ch1损失函数.py](e:/my_github/Cs_learning_of_liang/深度学习入门-小鱼书学习/u4_神经网络的学习/ch1损失函数.py#L35) 中，代码并没有每次都使用全部训练样本，而是随机抽取 `100` 个样本构成 batch。

这是因为：

- 全量数据计算太慢
- batch 方式更高效
- 后续梯度下降几乎都基于 mini-batch 来进行

### 实验现象及原因分析

这一节的输出主要用于观察 batch 的形状和标签形式。虽然没有复杂图像实验，但它奠定了后续训练代码的基础：

- 损失函数是可优化的目标
- batch 是后续所有训练代码的默认数据组织方式

## 2. 数值微分

### 实验目的

通过 notebook 形式理解导数、数值微分、梯度和梯度下降之间的关系。

### 实验原理

#### 1. 数值微分

导数的定义是函数变化率。数值微分的做法是：给自变量一个极小扰动，然后近似计算函数值变化率。

当前实现使用的是中心差分形式：

`(f(x+h) - f(x-h)) / (2h)`

这种写法比单边差分更稳定。

#### 2. 梯度

对于多变量函数，所有偏导数组成的向量就是梯度。  
梯度指出了函数值上升最快的方向，因此沿着负梯度方向更新参数，就有机会让损失函数下降。

#### 3. 梯度下降

梯度下降的更新规则为：

`x <- x - lr * grad`

其中 `lr` 为学习率。  
学习率太大可能越过最优点，学习率太小则会让收敛非常缓慢。

### 实验现象及原因分析

notebook 中的数值微分实验说明：

- `h` 不能取得太大，否则近似过粗
- `h` 也不能取得太小，否则会受到浮点精度影响
- 中心差分形式在精度和稳定性上更适合作为教学实现

这一部分虽然计算慢，但它为后面“梯度到底是什么”提供了可验证、可直观理解的基础。

## ch3 两层神经网络

### 实验目的

将前面章节的内容组合起来，构造一个可以实际学习参数的两层神经网络。

### 网络结构

当前网络结构可以写为：

`输入层 -> 隐藏层(sigmoid) -> 输出层(softmax)`

即：

1. `a1 = xW1 + b1`
2. `z1 = sigmoid(a1)`
3. `a2 = z1W2 + b2`
4. `y = softmax(a2)`

### 实验原理

#### 1. loss

`loss(x, t)` 通过前向传播得到预测 `y`，再和标签 `t` 计算交叉熵误差。

#### 2. accuracy

`accuracy(x, t)` 用于评估预测结果，但不参与梯度更新。

#### 3. numerical_gradient

这一版本的两层网络使用的是数值梯度，而不是误差反向传播。也就是说，每个参数的梯度都通过 `numerical_gradient()` 近似求出。

这也是为什么训练会非常慢：  
每个参数都要单独扰动一次并重新计算损失，参数一多，开销就会迅速爆炸。

### 实验现象及原因分析

这一实现能让损失逐步下降，说明“神经网络通过梯度更新参数”这一主线是成立的。  
但它也清楚暴露了一个问题：

- 数值微分虽然正确、直观
- 但用于真正训练网络时速度非常慢

这正是第五章要引入误差反向传播法的直接原因。

### 关于训练速度的说明

当前代码中两层网络的训练速度较慢，是很正常的。  
原因在于：

- `W1`、`b1`、`W2`、`b2` 参数很多
- 每个参数都要用数值微分重新计算损失
- 这比后面反向传播“一次反向算完所有梯度”要低效得多

因此，这一章更适合用来理解“学习的数学原理”，而不适合进行大规模、长时间训练。

## 本章中重要方法在实践中的使用时机

### 1. 交叉熵误差什么时候用

交叉熵误差适合：

- 分类任务
- 输出是概率分布的场景
- 希望明确衡量“预测概率和真实标签差多远”的情况

达到的效果：

- 作为一个连续可优化的目标函数
- 能配合 softmax 使用，支持梯度学习

### 2. mini-batch 什么时候用

mini-batch 几乎是现代神经网络训练的标准配置。  
适合：

- 训练数据量较大
- 希望在效率和稳定性之间折中

达到的效果：

- 提高计算效率
- 降低一次迭代的计算成本
- 为随机梯度下降及其变体提供基础

### 3. 数值微分什么时候用

数值微分适合：

- 学习导数和梯度概念
- 小规模函数实验
- 检查梯度是否实现正确

达到的效果：

- 提供一个直观的梯度计算方式
- 帮助验证更高效算法的正确性

不适合：

- 大规模训练
- 参数很多的网络

### 4. 梯度下降什么时候用

只要需要通过梯度更新参数，就会用到梯度下降思想。  
达到的效果是：沿着损失下降方向不断调整参数，使网络逐渐学会从数据中提取规律。

### 5. 两层网络什么时候用

两层网络适合：

- 入门学习神经网络训练流程
- 理解“前向传播 + 损失 + 梯度 + 更新”这一完整闭环
- 做小型实验验证

达到的效果：

- 结构简单、便于分析
- 足以说明神经网络学习的基本机制

但当任务更复杂时，需要更深网络以及更高效的梯度计算方法。
