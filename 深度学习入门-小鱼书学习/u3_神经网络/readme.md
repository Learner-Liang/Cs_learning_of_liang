# 第三章 神经网络

本目录对应《深度学习入门》中“神经网络”一章。本章的重点不在于训练网络，而在于理解一个已经给定参数的前馈神经网络如何完成从输入到输出的映射，并学会把这一过程写成可以实际运行的代码。

本目录中的主要代码如下：

- `ch1神经网络初步练习.py`：使用已经训练好的参数 `sample_weight.pkl`，对 MNIST 测试集执行前向推理，并计算识别准确率。
- `sample_weight.pkl`：预训练好的网络参数。

## 本章核心思想

神经网络可以被看作多层函数复合而成的系统。对于输入 `x`，若网络有三层权重，则前向传播可以写成：

`a1 = xW1 + b1`

`z1 = h(a1)`

`a2 = z1W2 + b2`

`z2 = h(a2)`

`a3 = z2W3 + b3`

`y = g(a3)`

其中：

- `h` 是隐藏层激活函数
- `g` 是输出层激活函数
- `W`、`b` 是各层参数

本章最重要的是理解：  
神经网络的“预测”本质上就是把输入数据依次通过若干线性变换和非线性变换，最终得到输出结果。

## ch1 神经网络初步练习

### 实验目的

使用已经训练好的网络参数，在 MNIST 测试集上完成前向推理，并理解以下几个问题：

- 神经网络如何逐层计算输出
- 为什么隐藏层和输出层使用的激活函数不同
- 为什么实际推理时经常采用 batch 处理

### 实验原理

#### 1. 三层前馈网络的前向传播

当前代码使用的网络结构可以理解为：

`输入层 -> 隐藏层1(sigmoid) -> 隐藏层2(sigmoid) -> 输出层(softmax)`

在 [ch1神经网络初步练习.py](e:/my_github/Cs_learning_of_liang/深度学习入门-小鱼书学习/u3_神经网络/ch1神经网络初步练习.py#L37) 到 [ch1神经网络初步练习.py](e:/my_github/Cs_learning_of_liang/深度学习入门-小鱼书学习/u3_神经网络/ch1神经网络初步练习.py#L47) 中，前向传播依次完成：

1. `a1 = xW1 + b1`
2. `z1 = sigmoid(a1)`
3. `a2 = z1W2 + b2`
4. `z2 = sigmoid(a2)`
5. `a3 = z2W3 + b3`
6. `y = softmax(a3)`

这正是一个典型的前馈神经网络预测流程。

#### 2. 为什么隐藏层用 sigmoid

sigmoid 的作用是引入非线性。  
如果每一层都只做线性变换，那么多层线性层叠加后本质上仍然只是一个线性变换，网络表达能力不会真正增强。

sigmoid 会把输入压到 `(0, 1)` 区间，虽然现在深层网络中常常更偏向使用 ReLU，但在入门阶段，sigmoid 有助于直观理解“非线性激活”的概念。

#### 3. 为什么输出层用 softmax

softmax 会把最后一层输出转换为一个概率分布，使每一类对应一个概率值，并满足所有概率之和为 1。  
因此它非常适合用于多分类问题，例如 MNIST 的 `0~9` 十分类任务。

#### 4. 为什么可以直接加载预训练参数

本章的目标是先理解“神经网络如何推理”，而不是立刻自己训练网络。因此代码通过 [sample_weight.pkl](e:/my_github/Cs_learning_of_liang/深度学习入门-小鱼书学习/u3_神经网络/sample_weight.pkl) 直接读取已经训练好的权重，再将测试集输入网络进行预测。

这样做的好处是：

- 省去训练成本
- 把注意力集中在前向传播流程
- 先理解“怎么用网络”，再过渡到后续章节的“怎么训练网络”

#### 5. 为什么采用 batch 处理

在 [ch1神经网络初步练习.py](e:/my_github/Cs_learning_of_liang/深度学习入门-小鱼书学习/u3_神经网络/ch1神经网络初步练习.py#L54) 到 [ch1神经网络初步练习.py](e:/my_github/Cs_learning_of_liang/深度学习入门-小鱼书学习/u3_神经网络/ch1神经网络初步练习.py#L61) 中，代码每次取 `100` 张图像一起送入网络。

这样做的原因是：

- `numpy` 对矩阵批量计算更高效
- 一次处理多个样本比一个个处理更快
- 这也是后面 mini-batch 学习的重要基础

### 实验现象及原因分析

运行该脚本后，可以得到模型在 MNIST 测试集上的准确率。说明：

- 只要参数已经训练好，神经网络就可以像一个普通函数一样直接做预测
- 同一套前向传播公式可以同时处理单样本和 batch 样本
- 使用 batch 预测不会改变数学本质，只是提高了计算效率

这一实验虽然没有画复杂图像，但它非常关键，因为它让我们第一次把“神经网络结构”真正落实成了可运行代码。

## 本章中重要函数在实践中的使用时机

### 1. sigmoid 什么时候用

sigmoid 适合：

- 作为入门阶段理解非线性激活函数的例子
- 在二分类输出层中输出概率
- 在一些门控结构中控制开关

达到的效果：

- 引入非线性
- 将输出压缩到 `(0, 1)` 区间
- 方便解释为概率或“激活程度”

需要注意的是：sigmoid 在深层网络中容易导致梯度消失，因此后续隐藏层常常更偏向使用 ReLU。

### 2. softmax 什么时候用

softmax 适合：

- 多分类任务的输出层
- 希望把输出转为类别概率分布的场景

达到的效果：

- 将原始得分转为概率
- 保证所有类别概率和为 1
- 便于后续与交叉熵损失配合使用

### 3. batch 推理什么时候用

batch 推理适合：

- 测试集评估
- 批量预测
- 希望提高矩阵计算效率的场景

达到的效果：

- 提升推理速度
- 更好地发挥 `numpy` 的向量化优势
